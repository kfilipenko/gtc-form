  - https://app.gtstor.com/game-chat.html
  - https://app.gtstor.com/chat-qa.html
- На первом этапе меняем только входную точку (главную), контент и навигацию.
- Без авторизации/регистрации.

## Концепт MVP (рекомендуемый)

**Формат:** одна landing-страница с лёгким 3D-эффектом + контентными секциями.

**3D-подача в MVP:**
- Фоновая 3D-сцена (частицы/карточки ролей/мягкий параллакс).
- 3D-эффект только декоративный, не блокирует UX.
- Для слабых устройств: авто-fallback на 2D фон.

## Информационная архитектура (одна страница)

1. Hero (первый экран)
2. Как это работает (3 шага)
3. Что внутри игры (роли/карты/режимы)
4. Медиа-блок (видео + фото)
5. FAQ (краткий)
6. Финальный CTA

## UX и визуальные принципы

- Главный акцент: быстрый путь к действию (чат) + контентное доверие (правила/медиа/FAQ).
- 3D не должен мешать читаемости и скорости.
- Mobile-first: CTA и текст в приоритете, 3D упрощается автоматически.

## Технический MVP-стек

Вариант 1 (самый быстрый):
- HTML/CSS/JS + Spline embed (или canvas-параллакс) для 3D-фона.

Вариант 2 (гибче на будущее):
- React + Three.js (React Three Fiber), если нужен дальнейший рост и сложная интерактивность.

Для MVP рекомендован Вариант 1.

## Критерии готовности MVP

- Одна страница опубликована и открывается без регистрации.
- Все CTA ведут на рабочие URL:
  - https://app.gtstor.com/game-chat.html
  - https://app.gtstor.com/chat-qa.html
- Контент включает: Hero, 3 шага, медиа, FAQ, финальный CTA.
- Есть 2D-fallback, если 3D отключён/не поддерживается.
