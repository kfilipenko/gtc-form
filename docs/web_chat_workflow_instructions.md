# Инструкция по развёртыванию GTC Sales Agent (Web Chat)

Ниже собраны шаги, которые помогут импортировать основной веб-чат и вспомогательный Web Search workflow в n8n, подключить необходимые секреты и локально проверить сценарий.

## 1. Файлы для импорта

| Workflow | Назначение | Путь к JSON |
| --- | --- | --- |
| **GTC Sales Agent - Web Chat** | Основная логика обработки запросов из веб-чата, включая аутентификацию, проверку подписок, логирование и вызов AI-агента. | `docs/workflows/GTC Sales Agent - Web Chat.json` |
| **Web Search OpenAI** | Подпроцесс, который AI-агент вызывает как tool для веб-поиска. | `docs/workflows/Web Search OpenAI.json` |

Импорт выполняется стандартным способом: `n8n → Workflows → Import from File`.

## 2. Переменные окружения и креды

| Переменная/credential | Где используется | Примечание |
| --- | --- | --- |
| `N8N_WEBHOOK_BASE` | Вебхук `POST /webhook/web-chat` (n8n ставит автоматически) | Убедитесь, что внешний URL проброшен через reverse proxy. |
| `DB_GTC` (Postgres credential) | Все `Postgres` узлы | Должен давать доступ к таблицам `user`, `auth_email`, `subscriptions`, `chat_log`. |
| `OPENAI_AZURE_CREDENTIAL` | Узел `AI Agent` | Профиль должен включать модели ProcureBot + fallback. |
| `TAVILY_API_KEY` | Подпроцесс `Web Search OpenAI` → узел `Tavily Search` | Добавьте переменную окружения в контейнер/процесс n8n: `export TAVILY_API_KEY=...`. |

## 3. Быстрая проверка после импорта

1. **Webhook URL:** в интерфейсе убедитесь, что у главного workflow включён Execute → Webhook testing. URL вида `https://<host>/webhook/web-chat`.
2. **Подготовьте данные:**
   ```json
   {
     "message": "Лучшие офферы на MacBook",
     "session_id": "demo-session-123",
     "metadata": {
       "email": "user@example.com",
       "fullName": "Demo User",
       "source": "web-demo"
     }
   }
   ```
3. **Отправьте запрос** через Postman/curl и убедитесь, что получаете `200` + JSON с ответом ассистента.
4. **Проверьте логирование:** записи должны появиться в `chat_log` с каналом `web`.
5. **Провалидируйте ветки ошибок:**
   - Отсутствующая подписка → `402`.
   - Неавторизованный пользователь → `401`.
   - Неформатируемый ответ агента → сообщение из узла `Format Response` + безопасная заглушка.
6. **Web Search tool:** отдельно выполните subworkflow через `Execute Workflow` с входом `{ "message": "цены на iPhone 15" }` и убедитесь, что выход содержит 1–5 отформатированных результатов.

## 4. Полезные заметки

- Основной workflow ожидает, что `session_id` присутствует в payload; если приходит пустым, генерируйте новый UUID на фронтенде.
- Вся история диалога ограничена последними 25 сообщениями (`History Window` узел). При необходимости увеличьте `LIMIT`.
- Code-узел `Sanitize AI Response` гарантирует отсутствие ссылок и markdown-таблиц. При обновлениях моделей следите за escape-последовательностями (`\\s`, `\\d` и т.д.).
- Подпроцесс поиска возвращает строку `output`, которую AI-агент подставляет в tool result; добавляйте новые поля (например, `answer`) при расширении логики.

После прохождения шагов выше система готова к e2e-тестам на стенде и дальнейшему релизу.
