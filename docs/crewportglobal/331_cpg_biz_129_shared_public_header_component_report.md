# CPG-BIZ-129 - Shared public header component report

- Project: CrewPortGlobal.com
- Date: 2026-06-10
- Status: Implemented
- Scope: public/application header consistency, CPG brand mark protection

## 1. Business Requirement

The top block that displays the CrewPortGlobal brand must have one design across portal pages. A future correction to this block must be made in one place and then propagate to the pages that use the common navigation/header runtime.

## 2. Implemented Result

The shared `crewportglobal-navigation.js` runtime now renders a standard `.site-header` component for ordinary public and application pages.

The shared header includes:

- brand link;
- protected `CPG` brand mark;
- protected `CrewPortGlobal` brand name;
- shared tagline slot;
- shared theme switcher;
- shared language selector;
- shared account/login area.

The header is generated by the common script before theme, account and language controls are rendered. This avoids maintaining many different manual header copies across HTML pages.

## 3. Brand Protection

The `CPG` brand mark is marked as non-translatable and normalized by the shared runtime.

Specialized workbench headers that do not use `.site-header` are not replaced because they contain local workflow controls, but their brand marks are also normalized to `CPG` and marked as non-translatable.

Covered brand-mark classes:

```text
.brand-mark
.cabinet-brand-mark
.shortlist-brand-mark
.registry-brand-mark
.matching-brand-mark
.team-mark
.admin-mark
```

## 4. CSS Result

The shared CSS now keeps the brand mark stable:

```text
width: 48px
height: 48px
flex-basis: 48px
white-space: nowrap
overflow: hidden
translate="no"
```

This prevents the previous issue where the mark could be machine-translated or wrapped into multiple lines.
