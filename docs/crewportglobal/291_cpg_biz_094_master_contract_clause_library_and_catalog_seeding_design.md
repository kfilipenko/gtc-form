## 1. Purpose

This document converts the approved CPG-BIZ-093 master-contract standard into a practical clause library and catalog seed.

The purpose is to prepare a complete seafarer / shipowner master contract model where:

1. fixed clauses are stable and versioned;
2. variable conditions are selected through embedded fields inside the Contract Agreement Workspace;
3. every variable states whether it is `single`, `multiple`, `linked_record`, `computed`, `date`, `number`, `money`, `text_controlled` or `document_reference`;
4. party approval/signature applies to the populated Contract Agreement Workspace before the contract-generation script creates the final document;
5. the generated contract is based on verified seafarer data, verified shipowner/employer data, verified vessel data, the approved master template and the approved workspace values.

This document is not legal advice and is not the final production agreement. It is the controlled drafting baseline for maritime legal review and future implementation.

## 2. Legal And Practice Orientation

The clause library is aligned with these official orientation sources:

1. ILO Maritime Labour Convention, 2006 - seafarer employment agreements, wages, hours of work/rest, leave, repatriation, medical care, accommodation, complaint handling and recruitment/placement controls.
2. ILO NORMLEX MLC text - Regulation 2.1 and Standard A2.1 for seafarers' employment agreements.
3. UK MCA MGN 477 (M) Amendment 5 - practical SEA information and agreement guidance.
4. UK MCA MGN 478 (M) Amendment 1 - wages guidance.
5. UK MCA MGN 479 (M) Amendment 1 - repatriation guidance.

These sources support the structure, but the production master agreement must still be reviewed as a template version. Ordinary generated instances should not be re-drafted if they use only approved clauses and approved variable catalogs.

Source URLs:

1. ILO Maritime Labour Convention, 2006 page - `https://www.ilo.org/international-labour-standards/maritime-labour-convention-2006`
2. ILO MLC, 2006 consolidated text and amendments - `https://www.ilo.org/resource/other/maritime-labour-convention-2006-amended-including-2022-amendments`
3. UK MCA MGN 477 (M), Amendment 5 - `https://www.gov.uk/government/publications/mgn-477-m-amendment-5-mlc-2006-seafarers-employment-agreements/mgn-477-m-amendment-5-mlc-2006-seafarers-employment-agreements`
4. UK MCA MGN 478 (M), Amendment 1 - `https://www.gov.uk/government/publications/mgn-478-m-maritime-labour-convention-2006-seafarers-wages/mgn-478-m-amendment-1-maritime-labour-convention-2006-seafarers-wages`

## 3. Contract Generation Model

The approved generation model is:

```text
verified seafarer record
+ verified shipowner/employer record
+ verified vessel record
+ approved master agreement version
+ approved Contract Agreement Workspace with embedded condition fields
= generated contract instance
```

The Contract Agreement Workspace is the place where the parties approve commercial and voyage-specific variables in the context of the clauses they affect.

The generated contract must not contain unapproved free-text legal clauses.

## 4. Choice Type Legend

| Choice type | Meaning |
|---|---|
| `single` | One value must be selected from an approved catalog. |
| `multiple` | One or more values may be selected from an approved catalog. |
| `linked_record` | Value is imported from a verified platform record. |
| `computed` | Value is generated by the system from approved records or selected terms. |
| `date` | Date value with validation. |
| `number` | Numeric value with units or limits. |
| `money` | Amount plus currency. |
| `text_controlled` | Short controlled text allowed only in a defined field, not a legal clause rewrite. |
| `document_reference` | Linked uploaded/verified document or formal document reference. |
| `signature` | Party approval/signature event. |

This agreement is entered into between the seafarer and the shipowner/employer identified in the approved populated Contract Agreement Workspace.

The shipowner/employer confirms that the representative signing or approving this agreement has authority to bind the shipowner/employer for the selected vessel and voyage.

Unless a separate signed agreement states otherwise, CrewPortGlobal is not the seafarer's employer and does not replace shipowner/employer obligations for employment, wages, safe working conditions, joining travel, repatriation, medical care or applicable maritime labour compliance.

## 8. Contract Workspace Guard

The Contract Agreement Workspace cannot be approved unless:

1. employer/company is verified or explicitly accepted by control exception;
2. representative authority exists;
3. vessel context exists;
4. seafarer identity and rank are verified enough for contract preparation;
5. wage amount, currency, payment frequency and payment method are explicit;
6. joining date/place and joining responsibility are explicit;
7. return destination, return responsibility and return transport mode are explicit;
8. required documents are uploaded or explicitly excepted;
9. `to_be_agreed` is resolved before signature unless a controlled exception is approved;
10. master agreement version and catalog version are approved.
