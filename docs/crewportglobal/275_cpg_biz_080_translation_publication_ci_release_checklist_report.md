## 1. Цель

Цель этапа - подключить утвержденный translation publication guard к стандартной release-последовательности и CI workflow.

До этого были готовы:

1. controlled runtime bundle publication command;
2. build-controlled `publication_version`;
3. read-only publication guard.

На этом этапе эти элементы объединены в release checklist, чтобы изменение переводов не проходило мимо проверок.

## 2. Local Release Checklist

Добавлена команда:

```bash
npm run check:cpg-i18n-release
```

Команда выполняет:

1. `npm run publish:cpg-i18n-runtime-bundle`;
2. `npm run check:cpg-i18n-publication-guard`;
3. `npx playwright test tests/crewportglobal-homepage-language.spec.ts --config=playwright.crewportglobal.config.ts`.

Это локальный аналог CI release-check для machine-localization publication.

## 3. CI Workflow

Добавлен workflow:

```text
.github/workflows/crewportglobal-i18n-publication.yml
```

Workflow запускается при:

1. pull request с изменениями translation/public/runtime paths;
2. push в `main` с такими изменениями;
3. manual `workflow_dispatch`.

CI sequence:

1. checkout;
2. setup Node.js 20;
3. install `php-cli` and `postgresql-client`;
4. install npm dependencies;
5. install Playwright Chromium;
6. run `npm run publish:cpg-i18n-runtime-bundle`;
7. run `npm run check:cpg-i18n-publication-guard`;
8. run generated-artifact diff check;
9. run focused browser regression.

CI uses a PostgreSQL service because the focused CrewPortGlobal browser regression starts the local PHP backend and applies the current migrations through `playwright.crewportglobal.config.ts`.

## 4. Generated Artifact Diff Check

CI checks that publication artifacts generated by the workflow are committed.

The check runs after publication and before Playwright report generation:

```bash
git diff --exit-code -- \
  projects/crewportglobal/i18n/runtime-bundle/crewportglobal-machine-translations.js \
  projects/crewportglobal/i18n/runtime-bundle/manifest.json \
  projects/crewportglobal/public/assets/crewportglobal-machine-translations.js \
  projects/crewportglobal/public
```

If source/catalog/public HTML changes require a new runtime bundle version but the generated files were not committed, CI fails.

## 5. Safety Boundaries

This stage does not change:

1. backend code;
2. database schema;
3. production runtime configuration;
4. Google provider credentials;
5. browser provider boundary;
6. rule that user-entered form values are not translated.

The CI workflow does not call Google Translate. It uses existing committed cache/bundle artifacts.
