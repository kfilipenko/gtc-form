## 1. Цель

Цель этапа - перестроить главную страницу из объяснительной страницы в продающий входной экран с двумя понятными сценариями:

1. моряк хочет создать профиль и найти контракт;
2. судовладелец / работодатель хочет разместить экипажную заявку.

Ключевое уточнение Project Owner зафиксировано в реализации: профиль моряка или рабочая форма работодателя открываются только после регистрации физического лица / участника платформы. Поэтому главная страница ведет не сразу в рабочую анкету, а в регистрацию с предвыбранной ролью.

## 2. Реализованная пользовательская логика

### 2.1 Главная страница

Главная страница теперь показывает:

1. результатный оффер вместо описания внутренней технологии;
2. два явных сценария входа:
   - `I am a seafarer`;
   - `I need crew`;
3. один основной CTA для каждого сценария;
4. короткий trust-блок:
   - отсутствие рекрутинговых сборов для моряков;
   - проверка данных работодателя;
   - прозрачный процесс и поддержка;
5. компактный блок партнеров и коротких историй для будущего social proof;
6. статистику реестра только если в системе есть ненулевые данные.

Нулевые счетчики и длинные операционные пояснения убраны из первого экрана.

### 2.2 Ролевая регистрация

Ссылки с главной страницы:

```text
/register/?role=seafarer
/register/?role=employer
```

Страница регистрации трактуется как регистрация участника платформы, а не как заполнение профессионального профиля.

После успешной регистрации:

| Роль | Следующий шаг |
|---|---|
| `seafarer` | `/create-profile/?draft_id=...` |
| `employer` | `/post-vacancy/?draft_id=...` |

Таким образом соблюдается утвержденный порядок:

```text
регистрация физического лица / участника платформы
-> определение роли
-> переход к профильной анкете или заявке
```

## 4. Проверка

### 4.1 Проверка встроенных скриптов

```bash
node - <<'NODE'
const fs = require('fs');
for (const file of [
  'projects/crewportglobal/public/index.html',
  'projects/crewportglobal/public/register/index.html',
]) {
  const html = fs.readFileSync(file, 'utf8');
  const scripts = Array.from(html.matchAll(/<script(?:\s[^>]*)?>([\s\S]*?)<\/script>/g))
    .map((match) => match[1])
    .filter((script) => script.trim());
  scripts.forEach((script) => new Function(script));
  console.log(`${file}: checked ${scripts.length} inline script(s)`);
}
NODE
```

Результат: встроенные скрипты главной страницы и страницы регистрации успешно разобраны.

## 5. Контролируемые ограничения

1. Блок партнеров пока демонстрационный. Публикация реальных партнеров из базы данных, логотипы и флаг `publish_on_homepage` еще не реализованы.
2. События аналитики конверсии еще не добавлены.
3. Отзывы и кейсы пока являются структурой для будущего наполнения, а не подтвержденными публичными отзывами.
4. Визуальная финальная правка главной страницы должна продолжаться после ручного просмотра Project Owner.
